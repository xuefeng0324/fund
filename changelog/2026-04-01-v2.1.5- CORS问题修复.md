# v2.1.4 & v2.1.5 - GitHub Pages CORS 问题修复

**日期**：2026-04-01

## 问题背景

部署到 GitHub Pages 后，基金数据无法正常加载，vConsole 显示请求没有发出。

## 排查过程

1. 添加 vConsole 调试工具查看日志
2. 发现请求卡在 `fetchFundBasicInfo` 函数
3. 测试发现 `fundmobapi.eastmoney.com` 接口返回的是纯 JSON，不是 JSONP 格式

## 根本原因

- `fundmobapi.eastmoney.com` 接口**不支持 JSONP**
- 即使添加 `jsonCallBack` 参数，返回的仍然是纯 JSON：`{"Datas":[...]}`
- JSONP 回调函数永远不会被触发，导致请求卡住

## 解决方案

| 接口 | 特性 | 调用方式 |
|------|------|---------|
| `fundmobapi.eastmoney.com` | 支持 CORS | fetch |
| `push2.eastmoney.com` | 支持 JSONP（cb 参数） | JSONP |
| `fundgz.1234567.com.cn` | 支持 JSONP | JSONP |
| `fund.eastmoney.com/pingzhongdata` | 不支持 CORS | script 标签 |

## 技术变更

### v2.1.4
- `push2.eastmoney.com` 改用 JSONP 方式
- 移除自定义请求头避免触发 CORS 预检

### v2.1.5
- `fundmobapi.eastmoney.com` 改回 fetch 方式（该接口支持 CORS）
- 添加 vConsole 调试工具便于生产环境排查问题

## 经验总结

1. 不是所有 API 都支持 JSONP，需要测试验证
2. 判断接口是否支持 JSONP：直接访问带 `cb` 或 `callback` 参数的 URL，看返回格式
3. 支持 CORS 的接口可以直接用 fetch，无需 JSONP
