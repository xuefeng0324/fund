# v2.5.0 (2026-04-18)

**新功能 — 基金交易规则同步功能**

新增 GitHub Actions 定时任务，自动同步基金交易规则数据（买入确认日、卖出确认日）。

## 主要变更

- 新增 `scripts/fund-info-sync.cjs` Node.js 同步脚本
- 新增 `.github/workflows/fund-info-sync.yml` GitHub Actions 工作流
- 新增 `public/config/fund_info.json` 基金交易规则存储文件
- 新增 `docs/specs/2026-04-17-fund-info-sync-design.md` 设计文档
- 新增 `docs/superpowers/plans/2026-04-17-fund-info-sync-plan.md` 实现计划

## 功能说明

- **定时同步**：每天 17:00 UTC（北京时间次日 01:00）自动执行
- **手动触发**：GitHub Actions 页面可手动 Run workflow
- **数据源**：danjuanfunds.com 基金详情接口
- **同步策略**：仅同步 `fund_info.json` 中不存在的基金，已存在数据永不更新
- **错误处理**：请求失败的基金跳过，不阻塞其他基金

## 技术实现

- 使用 Node.js 内置模块（fs、path、https），无需额外依赖
- 串行请求避免限流，100ms 间隔
- 10 秒请求超时
- 成功获取的数据合并写入 `public/config/fund_info.json`
