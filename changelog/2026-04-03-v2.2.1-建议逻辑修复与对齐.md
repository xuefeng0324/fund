# v2.2.1 - 建议逻辑修复与对齐 Python 脚本

## 概述

对比原始 Python 脚本 (1_fund_monitor.py)，修复了从 Python 改写为 JS 过程中引入的多个逻辑 Bug，确保买卖建议系统与 Python 版本行为一致。

## 变更详情

### useAdvice.js - 核心逻辑重写

**Bug 修复：gszzl（估算涨跌幅）未传递**
- loadAdvice 调用 getTradeAdvice 时未传入 gszzl，导致止盈、所有买入规则永远不触发
- 新增 fundsMap 参数，从实时基金数据中提取 GSZZL 传入建议计算
- 这是影响最大的 Bug，修复前大部分基金只会显示"持有"或"清仓"

**Bug 修复：持仓/空仓视角合并为单次计算**
- Python 脚本分别以 has_position=True 和 has_position=False 两次调用建议函数，再合并结果
- JS 改写时简化为一次调用同时分配 holdAction/flatAction，但每条规则提前 return 导致空仓视角丢失可能的买入信号
- 修复为对齐 Python 的双次独立计算 + mergeAdvice 合并模式

**Bug 修复：买入前提条件遗漏**
- Python 中买入需要同时满足：跌幅>=1% 且 周K在60线上方
- JS 中只检查了跌幅>=1%，遗漏了"周K在60线上方"条件
- 在熊市（周K跌破60线）时不应给出买入建议

**功能补全：空仓30线上方观望**
- Python 中空仓视角如果日K在30线上方，返回"观望，暂不买入"（不追高）
- JS 中完全缺失此规则

**功能补全：反弹减仓提醒**
- Python 中如果之前触发止损（跌破30线），后来反弹回30线上方且有涨幅，提醒"反弹减仓"
- JS 中缺失此规则

**功能补全：KDJ prevJ 传递**
- 新增 kdj_prev_j 字段，用于 KDJ 低位回升（J 穿越 20）判断

### kdj.js - 周线 ISO 周算法修复

**Bug 修复：groupWeeklyLast 跨年周归类错误**
- 原算法使用自定义公式近似 ISO 周，在跨年周（如 2025-12-29 属于 ISO 2026-W01）会归类到错误的年-周桶中
- 修复为标准 ISO 周算法：先找周四确定 ISO 年，再计算 ISO 周数
- 与 Python 的 date.isocalendar() 行为一致

### App.vue - 传递 funds 数据

- loadAdvice 调用时构建 fundsMap（code -> 基金实时数据映射）
- 确保每只基金的 GSZZL（估算涨跌幅）能被传入建议计算

## 涉及文件

- `src/composables/useAdvice.js` - 核心建议逻辑重写
- `src/utils/kdj.js` - 周线聚合算法修复
- `src/App.vue` - 传递实时数据到建议计算
