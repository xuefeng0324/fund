# v2.2.2 - 净值更新状态判断与 [已更新] 标签显示

**发布日期：** 2026-04-03

## 新增功能

### 1. T+1/T+2 基金类型判断

支持自动识别 T+2 基金类型：
- **QDII 基金**：基金名称包含 `(QDII)` 标识
- **香港互认基金**：基金代码以 `968` 开头

### 2. 净值更新状态判断

| 基金类型 | 判断标准 | 说明 |
|---------|---------|------|
| **T+1 基金** | `gztime == today` | 估值时间是今天算已更新 |
| **T+2 基金** | `gztime == yesterday` | 估值时间是昨天算已更新 |

### 3. [已更新] 标签显示

当基金净值已更新时，在基金名称后显示 `[已更新]` 标签：
- PC 端表格：名称列显示
- 移动端卡片：名称区域显示
- 全端一致显示

## 技术实现

### API 层 (`src/api/funds.js`)

新增函数：
- `yesterdayStr()` - 获取昨天日期字符串
- `extractDateFromGztime(gztime)` - 从 GZTIME 提取日期
- `isT2Fund(code, shortName)` - 判断是否为 T+2 基金
- `isNavUpdated(gztime, isT2)` - 判断净值是否更新

修改函数：
- `buildResults()` - 添加 `isNavUpdated` 和 `isT2` 字段到结果对象
- `fetchNoEstimateFunds()` - 添加无估值基金的更新状态判断

### 组件层 (`src/components/FundTable.vue`)

修改函数：
- `getFundName()` - 根据 `isNavUpdated` 字段添加 `[已更新]` 标签
