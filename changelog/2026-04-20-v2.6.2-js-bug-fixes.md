# v2.6.2 — JS 代码审查修复

**日期**: 2026-04-20

## 背景

对全项目 JS 文件进行两轮系统性代码审查，修复发现的 bug，删除无用函数，提升代码稳定性和可读性。

---

## 修复详情

### useAdvice.js

**1. 周线日期 UTC 偏移（高）**

- **问题**: `new Date(it.x).toISOString().slice(0, 10)` 将日期转为 UTC，在 UTC- 时区会导致日期错位一天，影响周线聚合和后续所有技术指标计算
- **修复**: 改用 `dayjs(it.x).format('YYYY-MM-DD')`，与 kdj.js 行为一致，统一本地时间解析
- **影响**: 周KDJ、MA30/MA60周线、波段心法买卖信号

**2. NaN 被误判为有效涨跌幅（中）**

- **问题**: `z != null` 对 `NaN` 返回 `true`，导致 `z > 0`、`z <= -1` 等后续比较静默失败，买入/止盈条件永远不触发
- **修复**: `parseFloat(gszzl)` 结果用 `Number.isFinite()` 检验，NaN 归为 null

---

### useConfig.js

**3. computed ref 被直接赋值（高）**

- **问题**: `fundCodes` 是 `computed(() => Object.values(fundGroups.value).flat()...)`，第 62 行 `fundCodes.value = cached.fundCodes` 会覆盖 getter 函数，使 `fundGroups` 变化时 `fundCodes` 不再自动重算
- **修复**: 删除该行，只保留 `fundGroups.value = cached.fundGroups`，让 computed 正确派生

---

### useFunds.js

**4. loading 状态在异步完成前关闭（高）**

- **问题**: `forEach` + fire-and-forget 模式下，`finally { loading.value = false }` 在所有 `fetchWithRetry` 开始时即触发，此时数据还未返回
- **修复**: 改为 `await Promise.allSettled(missing.map(code => fetchWithRetry(code)))`，确保所有请求完成后再关闭 loading
- **附带改善**: `fetchWithRetry` 提取为具名函数，接受 code 参数，逻辑更清晰

---

### funds.js

**5. JSONP 请求合并丢失原始 Promise（高）**

- **问题**: 相同 code 重复请求时，用新的 `resolve/reject` 覆盖旧的，原始 Promise 永远不会 resolve/reject，造成内存泄漏
- **修复**: `pendingRequests` 存储 `resolvers: [{ resolve, reject }]` 数组，重复请求追加而非覆盖，响应时遍历通知所有等待方

**6. 删除未使用的导出函数**

- `buildResults` — 曾用于批量接口结果构建，接口移除后成为死代码
- `fetchNoEstimateFunds` — 同上
- `todayStr` — 仅被 `buildResults` 使用，随之删除

---

### github.js

**7. `unescape` 废弃导致 Unicode 编码错误（中）**

- **问题**: `btoa(unescape(encodeURIComponent(...)))` 中 `unescape` 已废弃，对含有中文的 JSON 可能产生错误的 base64 编码
- **修复**: 抽取 `jsonToBase64(obj)` 辅助函数，使用 `TextEncoder` 逐字节转换，正确处理任意 Unicode 内容

**8. token 为空时发送空 Authorization header（中）**

- **问题**: `token ? \`token ${token}\` : ''` 在 token 为空时发送 `Authorization: `（空值），暴露未认证状态
- **修复**: 抽取 `buildHeaders(token)` 辅助函数，token 为空时完全省略该 header

---

### kdj.js

**9. 日期字符串本地时区解析（中）**

- **问题**: `new Date(date)` 对 `'YYYY-MM-DD'` 格式按 UTC 解析，UTC- 时区下 `getDay()` 返回前一天的星期，影响 ISO 周计算和周线聚合
- **修复**: 改用 `dayjs(date).toDate()`，dayjs 默认本地时间解析

**10. checkMainRise 除零保护（中）**

- **问题**: `prevHigh == null` 检查不包含 `prevHigh === 0`，导致 `(latest - 0) / 0` 产生 Infinity，涨幅判断失效
- **修复**: 条件改为 `prevHigh == null || prevHigh <= 0`

**11. Math.min/max 展开栈溢出（中）**

- **问题**: `Math.min(...window)`、`Math.max(...closes)` 等展开操作在数组较大时可能超出调用栈限制
- **修复**: 全部改为 `Array.reduce((a, b) => a < b ? a : b)` 形式，无展开限制
